# CVM 多可用区创建需求文档

## 1. 需求背景

当前 CVM 实例申请页在用户完成配置后，只能通过底部数量批量创建同配置实例，无法表达“同一套配置分布到多个可用区分别创建”的场景。

SCLB 高可用场景下，需要调用 CVM 接口分别在 A、B 可用区各创建一个实例。现有机制下，用户只能拆成多次申请，操作成本高，也不利于强化高可用部署意识。

本次需求目标是在尽量保持页面现状不变的前提下，支持用户在 1 张申请单内完成多个可用区实例分布配置。

## 2. 需求范围

### 2.1 本期支持

1. 仅支持同一机房下多个可用区创建。
2. 多可用区之间仅允许“可用区”和“实例数量”不同，其余配置保持一致。
3. 网络继续为系统分配。
4. 1 张申请单内包含多个可用区分布。
5. 只针对系统盘做可用区划分，数据盘不做划分。
6. 金山云、亦庄支持本期新增的多可用区创建能力。

### 2.2 本期不支持

1. 跨机房创建。
2. 不同可用区使用不同镜像、规格、磁盘、网络等差异化配置。
3. 创建结果按可用区展示独立状态。
4. 媒体机房不纳入本期多可用区改造范围。

## 3. 页面改动

## 3.1 实例申请页

在“机房”下新增“创建方式”，提供两种模式：

- 单可用区创建
- 多可用区创建

默认值：

- 默认机房：金山云
- 默认创建方式：单可用区创建
- 默认可用区：B 区

### 单可用区创建

1. 保持原有可用区按钮形式。
2. 底部数量输入框保持可编辑。
3. 在可用区下方保留提示文案：

`只针对系统盘做可用区划分，数据盘不做划分。`

4. 当数量大于 1 且当前机房支持多可用区时，在底部数量右侧位置显示引导文案：

`如需高可用部署，建议切换为多可用区创建`

其中“多可用区创建”为可点击入口，点击后切换至多可用区模式。

### 多可用区创建

1. 原“可用区”按钮区域替换为“可用区分布”表格。
2. 当前原型中可用区分布固定为 A/B 两个可用区，不再支持新增或删除分布项。
3. 每个可用区分别填写实例数量。
4. 底部数量入口保留，但改为只读，总数自动汇总。
5. 底部数量右侧文案调整为：

`请在上方可用区中调整实例数量`

### 媒体机房

1. 媒体机房仍可选中。
2. 选中媒体机房后，不展示“创建方式”。
3. 媒体机房保持现有单可用区申请方式，不支持本期多可用区能力。
4. 媒体机房下不展示高可用引导文案。

## 3.2 申请确认页

在保持原有布局基础上，仅增加与多可用区相关的信息展示：

1. 在公共配置区展示：
   - 创建方式
   - 总实例数
2. 新增“可用区分布”区块，展示：
   - 机房
   - 可用区
   - 实例数量
3. 不展示：
   - 状态
   - 备注
   - 额外说明文案

## 3.3 申请单详情页

在保持原有布局基础上，仅增加与多可用区相关的信息展示：

1. 在公共配置区展示：
   - 创建方式
   - 总实例数
2. 新增“可用区分布明细”区块，展示：
   - 机房
   - 可用区
   - 申请数量
3. 不展示：
   - 创建状态
   - 备注

## 4. 交互说明

## 4.1 创建方式切换

### 单可用区创建 -> 多可用区创建

1. 隐藏原可用区按钮区域。
2. 展示可用区分布表格。
3. 默认展示 A/B 两个可用区。
4. 底部数量自动汇总为各分布项实例数之和，且不可编辑。

### 多可用区创建 -> 单可用区创建

1. 隐藏可用区分布表格。
2. 恢复原可用区按钮区域。
3. 默认保留 1 个当前可用区。
4. 底部数量恢复为可编辑。

## 4.2 数量联动

### 单可用区创建

1. 底部数量表示当前所选可用区内创建的实例数量。
2. 用户可直接修改底部数量。
3. 当数量大于 1 时，在底部数量右侧位置展示高可用引导。

### 多可用区创建

1. A 区、B 区分别填写实例数量。
2. 底部数量自动汇总所有分布项的实例数。
3. 任一分布项数量变更后，底部总数量同步更新。
4. 底部数量右侧不再显示“可直接编辑”，改为提示用户到上方调整数量。

## 4.3 机房切换

### 金山云 / 亦庄

1. 支持创建方式切换。
2. 默认可用区为 B 区。
3. 支持高可用引导文案。

### 媒体机房

1. 隐藏创建方式。
2. 默认回到单可用区形态。
3. 不展示高可用引导文案。

## 5. 校验规则

1. 单可用区模式下，必须选择 1 个可用区。
2. 多可用区模式下，A/B 两个可用区都展示。
3. 每个分布项实例数量必须大于 0。
4. 所有分布项必须属于当前所选机房下的可用区。

## 6. 展示与提交口径

1. 整体仍为 1 张申请单。
2. 公共配置展示一次。
3. 可用区分布作为独立区块展示。
4. 总实例数按分布明细汇总计算。

## 7. 验收标准

1. 默认进入申请页时，机房为金山云，创建方式为单可用区创建，可用区为 B 区。
2. 单可用区模式下，保留原有可用区选择方式和可编辑数量输入。
3. 单可用区模式下，显示提示文案：
   `只针对系统盘做可用区划分，数据盘不做划分。`
4. 单可用区模式下，当数量大于 1 时，在底部数量右侧显示高可用引导文案。
5. 多可用区模式下，显示固定 A/B 两区分布。
6. 多可用区模式下，底部数量自动汇总且不可编辑。
7. 多可用区模式下，底部数量右侧显示：
   `请在上方可用区中调整实例数量`
8. 媒体机房可选，但不展示创建方式，不展示高可用引导。
9. 申请确认页新增可用区分布区块，不展示状态、备注、额外说明。
10. 申请单详情页新增可用区分布明细区块，不展示创建状态、备注。
