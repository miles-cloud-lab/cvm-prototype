# PRD - CVM 多可用区申请

## 需求背景

当前 CVM 申请页在用户完成配置后，只能通过底部数量控制批量创建同配置实例，无法表达“同一套配置在不同可用区分别创建实例”的场景。SCLB 做高可用时，需要调用 CVM 接口在 A、B 可用区各创建一个实例，现有机制只能分别提交两次申请，成本高且不利于强化用户高可用意识。

## 需求目标

1. 在 CVM 平台支持一次申请多个可用区实例。
2. 保持用户现有使用习惯，页面结构尽量不变。
3. 明确实例在可用区之间的分布，提升高可用部署意识。

## 方案边界

1. 只支持同一机房下多个可用区。
2. 多可用区之间仅可用区和实例数量不同，其余配置保持一致。
3. 网络继续统一为系统分配。
4. 1 张申请单承载多个可用区分布。
5. 只针对系统盘做可用区划分，数据盘不做划分。

## 页面改动

### 实例申请页

1. 在“机房”下新增“创建方式”，支持“单可用区创建”和“多可用区创建”。
2. 单可用区时，沿用原有可用区按钮和底部数量输入。
3. 多可用区时，在原可用区区域替换为可用区分布表格，每行配置一个可用区及数量。
4. 保留底部数量区域，但改为自动汇总展示。

### 申请确认页

1. 保持原有公共配置区块。
2. 新增“可用区分布”区块，展示机房、可用区、实例数量。
3. 不增加状态、备注等额外字段。

### 申请单详情页

1. 保持原有申请信息、实例公共配置、审核信息布局。
2. 新增“可用区分布明细”区块，展示机房、可用区、申请数量。
3. 不增加创建状态、备注等额外字段。

## 目标用户

- 业务线用户
- SCLB 用户
